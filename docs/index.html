Restaurant Order App

Overview

This is a restaurant order-tracking app that enables the customer to create their own order, in this case a burger; read the status of the order; update the contents and/or status of the order; and finally, delete the order - in other words, all of the CRUD operations are represented. This program was developed by Steven Bowler for the purpose of gaining experience developing a full-stack app employing best-practices for executing the Model-View-Control (MVC) software development pattern. Additionally, the project provides this programmer the opportunity develop skills in creating and deploying a functional full-stack app on Heroku, integrating Express-Handlebars, ExpressJS, and MySQL. Enjoy.

To use the app click burger

  1. To create a burger: Under the Create a Burger heading, enter the name of the burger you desire in the text box and click the Create Burger button.
  2. The name of the burger just created will be read into the bottom of the To Be Devoured list.
  3. To update the name of the burger: Under the Update a Burger heading,
    1. From the dropdown: select the burger name you wish to update
    2. In the textbox to the right of the dropdown: type in the new burger name.
    3. Click the Update a Burger button.
  4. To update the status of any burger from To Be Devoured to Devoured: click the Devour Burger button to the right.
  5. To delete the burger: Under the Devoured heading: click the Delete Burger button to the right of the burger to be deleted.

Program Documentation

App executes the MVC software pattern in the following manner:

  1. Initiate server.js
  2. Serve index.handlebars with current burgers table from burgers_db in MySQL/Heroku JawsDB.
  3. User onclick event triggers one of four actions: Create Burger, Update Burger, Devour Burger, Delete Burger. See module public/assets/js/burgers. This makes the AJAX call with the appropriate route.
  4. public/assets/js/burgers JQuery based user input module requires controllers/burgers_controller.
  5. controllers/burgers_controller module is the router that requires models/burger to make burger specific calls to MySQL.
  6. models/burger module formats parameters in the correct MySQL query format and requires the generic MySQL functions in config/orm.
